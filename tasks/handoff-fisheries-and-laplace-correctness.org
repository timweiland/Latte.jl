* Open issue: fisheries tutorial framing is wrong

The current =fisheries_state_space.jl= uses a *non-centered*
parameterisation:

- Latent: =ε[1:T-1]= IID standard Normal (truly Gaussian prior, IID
  precision = =I=).
- Dynamics: =log B[t+1] = ode_step(log B[t], θ) + σ_proc · ε[t]=
  applied inside the likelihood.

This *fits Latte's LGM framework* (which requires a Gaussian latent
prior) but throws away the Markov structure: =log B[t]= depends
*cumulatively* on =ε[1:t-1]=, so the likelihood Hessian on =ε= is
*dense lower-triangular*, and so is the posterior precision.

Real TMB stock assessments (e.g. SPiCT — Pedersen & Berg 2017) use
the *centered* parameterisation:

- Latent: =log B[1:T]= directly.
- Process equation as a likelihood term: =log B[t] ~ Normal(log B[t-1]
  + drift(log B[t-1], θ), σ_proc)=.
- Joint Hessian on =log B= is *tridiagonal* — Markov structure
  preserved.

This is the source of TMB's scalability. Our non-centered approach
gives correct inference but not the sparse-Hessian story we sold in
the tutorial.

Sources I confirmed:
- TMB paper: Kristensen et al. 2016, /Journal of Statistical Software/ 70(5).
- SPiCT C++ source: =PARAMETER_VECTOR(logB)=, =ans -=
  dnorm(logBpred(i+1), logB(i+1), sqrt(dt(i))*sdb, 1)=. Tridiagonal
  Hessian on =logB=.
- TMB docs: "For latent variables only normal distributions should be
  used (otherwise the Laplace approximation will perform poorly)."

* The deeper question we punted on

Latte *can* take a non-Gaussian DPPL prior and Gaussianify it via
sparse-AD on the prior log-density (=_build_joint_sparse_ad_latent= in
=src/dsl/latent_prior.jl=). But it *linearises at =x_0 = zeros=*,
fixed at adapter-build time. For state-space Schaefer with
=log B ≈ log K ≈ 7=, linearising at =x = 0= gives prior-Hessian
values that are off by ~3 orders of magnitude from where the data
actually pulls the posterior. *The sparsity pattern is right
(tridiagonal), but the values are biased.*

TMB instead Newtons the *full joint* and re-evaluates the Hessian
at each iterate — equivalent to a *per-Newton-step re-linearisation*
of the prior. That's the missing piece in Latte's algorithm.

* Design doc handed off to GMRFs.jl

=local_latent_quadratic.org= at the root of
=~/.julia/dev/GaussianMarkovRandomFields/=. Self-contained, no Latte
references. Proposes:

- New extension point: =local_quadratic(m::LatentModel, x_ref; θ...)
  -> LocalLatentQuadratic= where =LocalLatentQuadratic= carries =(Q,
  h, logp_ref, x_ref)=.
- Default method for genuinely Gaussian latent models — ignores
  =x_ref= for =(Q, h)=, only uses it for =logp_ref=. *Zero behaviour
  change* on existing code.
- Custom override per non-Gaussian-prior model that re-linearises at
  =x_ref=.
- =gaussian_approximation= Newton loop calls =local_quadratic= per
  iterate, with damping + line search on the *exact* joint
  log-density and modified Cholesky for indefinite =J= during the
  search.
- At convergence, recover the undamped =J(x*)= for the Laplace
  covariance and the *exact* =log p(x*, y | θ)= for the marginal
  log-likelihood (the =logp_ref= field, which equals the prior at
  =x*= when =x_ref = x*=).

The math, API sketch, default method strategy, convergence guidance,
sparse-AD prep gotchas, acceptance criteria, and references are all
there.
